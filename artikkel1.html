Git

Git er et versjonkontroll-system som ble utviklet i 2005 av Linus Torvalds. Programvaren gjør det mulig for brukerne å se over hvem som har kodet hva, samt ha et oversikt over hvem som har gjort endringer på visse ting. Dette vil også si at det er fullt mulig å holde styr over hva som blir redigert (og hvem som redigerer hva) underveis i kodinga.


Github

Github er det samme som Git, bare at brukerne har en online tilgang til den. Github samme konsept som «word online», hvor flere kan skrive og redigere sammen på én fil, bare at det blir benyttet for koding i stedet for skriving. Dette betyr altså at brukerne kan laste opp både filer og mapper som kan eventuelt deles med venner og arbeidspartnere. Github inkluderer en rekke med nyttige ting, som blant annet; funksjonsforespørsler, oppgavebehandling og bug-sporing.


Fordeler med å bruke Git og hva jeg synes om det

Det finnes flere fordeler med å bruke Git. Det første som kommer på tanken er hvor effektivt Git kan være med tanke på et oppbevaringssted som ikke koster noe. Man har tilgang til en ekstra OneDrive, bare at det er for kodefiler! Det gjør at vi kan både spare plass på disken, samt ha en enkel og gratis tilgang til det på nettet. Som nevnt tidligere, så er det en stor fordel å bruke Git på grunn av dette tracking-systemet som gir deg et oversikt over blant annet hvilke endringer som har blitt gjort underveis i kodinga, og ikke minst - hvem som har gjort hvilke endringer. Personlig, så synes jeg at idéen om Git er strålende, og det gir oss kodere en god mulighet som er vanskelig å unngå. Det er en veldig fin idé å kode sammen med enten venner eller arbeidspartnere på denne måten, og det gjør det både lettere og gøyere for oss.
